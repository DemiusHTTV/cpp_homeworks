# Лекция 10: «Инкапсуляция» на примере игры «Четыре в ряд»

Папка с выполнением: `task_lektsiya10_encapsulation/`.

В этой реализации я повторил код и идеи из лекции: класс `Field` инкапсулирует *данные* (ячейки поля, очередь хода, победителя) и предоставляет *методы* (очистка, ход, проверка победы, печать).

## Что делает программа (смысл)

- Создаёт поле 7×6.
- Игроки (красный и жёлтый) ходят по очереди, вводя номер колонки `1..7`.
- Фишка «падает вниз»: занимает самую нижнюю свободную клетку выбранной колонки.
- После каждого хода программа проверяет, появился ли ряд длины 4 по горизонтали, вертикали или диагонали.
- Игра заканчивается победой одного из игроков либо ничьей при полном заполнении поля.

Ниже — **очень подробный синтаксический разбор** (что означает каждая строка/конструкция, почему так написано и как это называется в C++).

## `Field.h` — типы и интерфейс класса

### Код (с номерами строк)

```cpp
 1  #pragma once
 2
 3  #include <iosfwd>
 4
 5  // Содержимое ячейки игрового поля.
 6  // По лекции: EMPTY, RED, YELLOW.
 7  enum Cell
 8  {
 9      EMPTY,
10      RED,
11      YELLOW
12  };
13
14  // Размеры поля "Четыре в ряд" (7x6) по правилам из лекции.
15  constexpr int FIELD_WIDTH = 7;
16  constexpr int FIELD_HEIGHT = 6;
17
18  class Field
19  {
20  public:
21      // Конструктор: задаёт, кто ходит первым.
22      explicit Field(bool isRedFirst);
23
24      // Полная очистка состояния поля (включая победителя и очередь хода).
25      void clear(bool isRedFirst);
26
27      // Сделать ход в колонку 1..FIELD_WIDTH.
28      // Возвращает true, если ход выполнен, иначе false (ошибка/нельзя ходить).
29      bool makeTurn(int column);
30
31      // Проверка победы для заданного цвета.
32      bool isWon(bool red) const;
33
34      // Игра окончена: кто-то выиграл или поле заполнено.
35      bool isOver() const;
36
37      // Доступ к ячейке поля (i — колонка 0..FIELD_WIDTH-1, j — строка 0..FIELD_HEIGHT-1).
38      Cell getCell(int i, int j) const;
39
40      // Чья очередь сейчас ходить.
41      bool isRedTurnNow() const;
42
43      // Вывод поля в консоль.
44      void print() const;
45
46      // Вывод результата (победитель или ничья).
47      void printResult() const;
48
49  private:
50      // Двумерный массив ячеек (по лекции).
51      Cell cells[FIELD_WIDTH][FIELD_HEIGHT];
52
53      // Очередь хода (true — ходит красный).
54      bool isRedTurn;
55
56      // Победитель (EMPTY — пока нет победителя).
57      Cell winner;
58
59      // Внутренняя проверка победителя после хода.
60      void checkWinner();
61  };
```

### Построчно и по синтаксису

1. `#pragma once` — директива препроцессора (*preprocessor directive*). Говорит компилятору включать заголовок только один раз в одну единицу трансляции, чтобы избежать повторных объявлений. Альтернатива — include guards (`#ifndef/#define/#endif`).
3. `#include <iosfwd>` — подключение стандартного заголовка. `iosfwd` содержит *предварительные объявления* потоковых типов (`std::ostream`, `std::istream` и т.п.). Это ускоряет компиляцию заголовка (мы не тянем весь `<iostream>` без нужды).
7–12. `enum Cell { ... };` — **перечисление** (*enumeration*). Это пользовательский тип, принимающий одно из перечисленных значений.
9–11. `EMPTY`, `RED`, `YELLOW` — **перечислители** (*enumerators*). По умолчанию им присваиваются целые значения 0, 1, 2 (как в лекции объяснялось), но мы используем их как символические имена состояний ячейки.
15–16. `constexpr int ...` — **константное выражение времени компиляции** (*compile-time constant*). `constexpr` значит: значение известно на этапе компиляции. Это удобно для размеров массива `cells[...]` — размеры должны быть константами времени компиляции.
18–61. `class Field { ... };` — объявление **класса**. Класс объединяет данные и функции (методы) в единый тип.
20. `public:` — **спецификатор доступа**. Всё ниже становится доступно «снаружи» объекта (пользователю класса).
22. `explicit Field(bool isRedFirst);` — объявление **конструктора**. `explicit` запрещает неявные преобразования из `bool` в `Field`, то есть запрещает, например, случайный код `Field f = true;` (это называется защита от *implicit conversion*).
25. `void clear(bool isRedFirst);` — метод, который ничего не возвращает (`void`) и принимает `bool`. `bool` — встроенный логический тип.
29. `bool makeTurn(int column);` — метод, возвращающий `bool` (успех/неуспех). Параметр `int` — целое число (номер колонки).
32, 35, 38, 41, 44, 47. Суффикс `const` после скобок параметров (`... ) const`) — это **константность метода** (*const member function*). Она означает: метод не имеет права изменять поля объекта (кроме `mutable`). Это важно: «проверяющие» функции (`isOver`, `getCell`, `print`) не должны менять состояние поля.
49. `private:` — второй **спецификатор доступа**. Всё ниже скрыто от внешнего кода. Это и есть **инкапсуляция**: внешнему миру доступен интерфейс, но не внутренние детали хранения.
51. `Cell cells[FIELD_WIDTH][FIELD_HEIGHT];` — **статический массив фиксированного размера** (встроенный массив C/C++). Здесь 2D-массив (массив массивов). Мы храним состояние каждой ячейки.
54. `bool isRedTurn;` — поле класса: «кто ходит сейчас».
57. `Cell winner;` — поле класса: победитель. `EMPTY` означает «победителя нет».
60. `void checkWinner();` — закрытый метод: проверка победы. Снаружи он не нужен, поэтому прячется в `private`.

## `Field.cpp` — реализация методов

### Код (с номерами строк)

```cpp
  1  #include "Field.h"
  2
  3  #include <iostream>
  4
  5  Field::Field(bool isRedFirst)
  6  {
  7      clear(isRedFirst);
  8  }
  9
 10  void Field::clear(bool isRedFirst)
 11  {
 12      isRedTurn = isRedFirst;
 13      winner = EMPTY;
 14      for (int i = 0; i < FIELD_WIDTH; i++)
 15          for (int j = 0; j < FIELD_HEIGHT; j++)
 16              cells[i][j] = EMPTY;
 17  }
 18
 19  bool Field::makeTurn(int column)
 20  {
 21      if (winner != EMPTY || column < 1 || column > FIELD_WIDTH)
 22          return false;
 23
 24      int i = column - 1;
 25      for (int j = 0; j < FIELD_HEIGHT; j++)
 26          if (cells[i][j] == EMPTY)
 27          {
 28              cells[i][j] = isRedTurn ? RED : YELLOW;
 29              checkWinner();
 30              isRedTurn = !isRedTurn;
 31              return true;
 32          }
 33      return false;
 34  }
 35
 36  bool Field::isWon(bool red) const
 37  {
 38      return winner == (red ? RED : YELLOW);
 39  }
 40
 41  bool Field::isOver() const
 42  {
 43      if (winner != EMPTY)
 44          return true;
 45
 46      for (int i = 0; i < FIELD_WIDTH; i++)
 47          for (int j = 0; j < FIELD_HEIGHT; j++)
 48              if (cells[i][j] == EMPTY)
 49                  return false;
 50      return true;
 51  }
 52
 53  Cell Field::getCell(int i, int j) const
 54  {
 55      return cells[i][j];
 56  }
 57
 58  bool Field::isRedTurnNow() const
 59  {
 60      return isRedTurn;
 61  }
 62
 63  void Field::checkWinner()
 64  {
 65      const int DIR_NUMBER = 4;
 66      const int di[] = {1, 0, 1, 1};
 67      const int dj[] = {0, 1, -1, 1};
 68      const int WIN_LENGTH = 4;
 69
 70      for (int i = 0; i < FIELD_WIDTH; i++)
 71          for (int j = 0; j < FIELD_HEIGHT; j++)
 72          {
 73              Cell start = cells[i][j];
 74              if (start == EMPTY)
 75                  continue;
 76
 77              for (int dir = 0; dir < DIR_NUMBER; dir++)
 78              {
 79                  int length = 0, iline = i, jline = j;
 80                  while (++length < WIN_LENGTH)
 81                  {
 82                      iline += di[dir];
 83                      jline += dj[dir];
 84                      if (iline < 0 || iline >= FIELD_WIDTH || jline < 0 || jline >= FIELD_HEIGHT)
 85                          break;
 86                      if (cells[iline][jline] != start)
 87                          break;
 88                  }
 89                  if (length == WIN_LENGTH)
 90                  {
 91                      winner = start;
 92                      return;
 93                  }
 94              }
 95          }
 96  }
 97
 98  void Field::print() const
 99  {
100      using std::cout;
101
102      cout << \"\\n\";
103      for (int j = FIELD_HEIGHT - 1; j >= 0; j--)
104      {
105          cout << \"|\";
106          for (int i = 0; i < FIELD_WIDTH; i++)
107          {
108              char ch = '.';
109              if (cells[i][j] == RED)
110                  ch = 'R';
111              else if (cells[i][j] == YELLOW)
112                  ch = 'Y';
113              cout << ch << \"|\";
114          }
115          cout << \"\\n\";
116      }
117      cout << \" \";
118      for (int col = 1; col <= FIELD_WIDTH; col++)
119          cout << col << \" \";
120      cout << \"\\n\\n\";
121  }
122
123  void Field::printResult() const
124  {
125      using std::cout;
126
127      print();
128      if (winner == RED)
129          cout << \"Победил красный игрок.\\n\";
130      else if (winner == YELLOW)
131          cout << \"Победил желтый игрок.\\n\";
132      else
133          cout << \"Ничья.\\n\";
134  }
```

### Ключевые конструкции и почему так

1. `#include "Field.h"` — подключаем **свой заголовок** (кавычки значит «ищи сначала рядом/в путях проекта»).
3. `#include <iostream>` — подключаем стандартные потоки `std::cout`, `std::cin`. Угловые скобки — «ищи в стандартных путях».
5. `Field::Field(...)` — **определение метода класса** с оператором области видимости `::` (*scope resolution operator*). Он говорит: это конструктор, принадлежащий классу `Field`.
7. `clear(isRedFirst);` — вызов метода для инициализации объекта. По лекции конструктор просто вызывает `clear`.
12–13. Присваивания `=` — **оператор присваивания**.
14–16. Два `for` без фигурных скобок — синтаксис: если тело цикла — одна инструкция, скобки можно опустить. Это соответствует стилю слайдов лекции.
21–22. `if (...) return false;` — **ранний выход** (early return): если игра уже выиграна или колонка вне диапазона — ход запрещён.
24. `int i = column - 1;` — перевод из «человеческой нумерации 1..7» в «индексацию массива 0..6». Это важно, потому что массивы в C++ индексируются с нуля.
25–33. Поиск первой свободной ячейки снизу: цикл `j` идёт от `0` вверх. Мы договорились, что `j=0` — нижняя строка. Это согласовано с правилом «фишка падает вниз».
28. `isRedTurn ? RED : YELLOW` — **тернарный оператор** (*conditional operator*). Это выражение вида `условие ? значение1 : значение2`.
30. `isRedTurn = !isRedTurn;` — `!` это **логическое НЕ** (*logical NOT*), переключаем очередь хода.
36. `... const` — метод обещает не менять объект.
38. Снова тернарный оператор: подбираем, какой `Cell` соответствует параметру `red`.
43–50. Проверка окончания: если победитель уже есть — конец; иначе ищем свободную клетку. Если нашли — игра ещё идёт.
65–68. `const int ...` — **локальные константы функции** (не меняются в теле `checkWinner`).
66–67. `const int di[] = { ... };` — массив направлений: (1,0) вправо, (0,1) вверх, (1,-1) диагональ вверх-вправо, (1,1) диагональ вниз-вправо (в нашем представлении осей).
79. `int length = 0, iline = i, jline = j;` — объявление нескольких переменных одного типа в одной строке (разделяются запятыми).
80. `while (++length < WIN_LENGTH)` — **префиксный инкремент** `++length`. Сначала увеличиваем `length`, потом сравниваем. Это буквально как в слайде.
84–87. Проверка выхода за границы массива и проверка совпадения цветов.
89–93. Если нашли 4 подряд — фиксируем победителя в поле `winner` и выходим `return;`.
98–121. Печать поля: используем `char` (`'.'`, `'R'`, `'Y'`) как простой символ для консоли. Печатаем строки сверху вниз, чтобы визуально было как игровое поле.
123–134. `printResult` печатает итог: победитель или ничья.

## `main.cpp` — управление игрой (цикл ввода/вывода)

### Код (с номерами строк)

```cpp
 1  #include \"Field.h\"
 2
 3  #include <clocale>
 4  #include <iostream>
 5
 6  int main()
 7  {
 8      std::setlocale(LC_ALL, \"Russian\");
 9
10      Field field(true);
11      while (!field.isOver())
12      {
13          field.print();
14          std::cout << \"Ходит \" << (field.isRedTurnNow() ? \"красный\" : \"желтый\")
15                    << \" игрок, введите ход (1-\" << FIELD_WIDTH << \"):\\n\";
16
17          int column;
18          if (!(std::cin >> column))
19              return 0;
20
21          if (!field.makeTurn(column))
22              std::cout << \"Некорректный ход. Повторите ввод.\\n\";
23      }
24
25      field.printResult();
26      return 0;
27  }
```

### Построчно и по синтаксису

6. `int main()` — **точка входа** программы на C++. Возвращаемый тип `int` — код завершения (0 обычно означает «успешно»).
8. `std::setlocale(LC_ALL, "Russian");` — настройка локали C-библиотеки (для корректного вывода русских строк в некоторых окружениях). `LC_ALL` — макрос из `<clocale>`.
10. `Field field(true);` — создание объекта (экземпляра) класса `Field`. Это **автоматический объект** (живёт до конца блока `main`).
11. `while (!field.isOver())` — **цикл while**: повторяем, пока игра не закончена. `!` — логическое НЕ.
14–15. Оператор `<<` — **оператор вывода в поток** (*stream insertion operator*). Он перегружен в стандартной библиотеке для разных типов.
14. `(field.isRedTurnNow() ? "красный" : "желтый")` — тернарный оператор, выбираем строковый литерал `const char*` для сообщения пользователю.
17. `int column;` — объявление переменной.
18. `std::cin >> column` — **оператор ввода из потока** (*stream extraction operator*). Возвращает поток; в логическом контексте можно проверить, успешно ли считалось значение.
18–19. `if (!(std::cin >> column)) return 0;` — если ввод закончился или неверный (например, буква вместо числа), завершаем программу.
21–22. Если `makeTurn` вернул `false`, ход был невозможен (ошибка диапазона/колонка заполнена/игра уже завершена) — просим повторить.
25–26. Печатаем результат и возвращаем 0.

## Как это связано с «устройством C++» (важные понятия)

- **Единица трансляции**: `.cpp` файл после работы препроцессора (все `#include` вставлены) компилируется отдельно в объектный файл.
- **Разделение интерфейса и реализации**: в `Field.h` — объявления (что умеет класс), в `Field.cpp` — определения (как именно реализовано).
- **Инкапсуляция**: `private` скрывает детали хранения (`cells`, `winner`, `isRedTurn`), а `public` даёт безопасные методы.
- **Перечисления**: `enum Cell` делает состояния ячейки ограниченным набором значений, улучшает читаемость и снижает вероятность ошибок.
- **Const-корректность**: методы `... const` защищают объект от случайного изменения при «просмотрах».
